# Bliki: Interrogatory LLM

## Summary
Martin Fowler introduces the term "Interrogatory LLM" — a pattern where users interact with large language models primarily through questioning and interrogation rather than declarative commands. This represents a fundamental shift in how developers and knowledge workers approach AI-assisted problem solving.

## Why This Matters
The interrogatory approach changes the interaction paradigm from "tell the AI what to do" to "ask the AI what it knows." This matters because LLMs perform differently when prompted as knowledge sources versus execution engines — understanding this distinction helps users get better results.

## Key Points
- **Interrogatory vs. Imperative**: Asking "what should I do?" vs. commanding "do this"
- **Knowledge elicitation**: Extracting domain knowledge from the model before taking action
- **Iterative refinement**: Using questions to progressively narrow down solutions
- **Risk reduction**: Interrogation surfaces edge cases and assumptions before committing to implementation
- **Model self-awareness**: LLMs often provide better answers when asked to reason through problems step by step

## How to Use Interrogatory LLM Patterns

### Basic Interrogation Flow
1. **Initial question**: "What are the options for X?"
2. **Follow-up**: "What are the tradeoffs of option A vs option B?"
3. **Deep dive**: "What specific pitfalls should I watch for with option A?"
4. **Decision**: Based on accumulated knowledge, choose a path
5. **Implementation**: Now use imperative commands with informed context

### Example: Architecture Decision
```
Q: "I need to choose between gRPC and REST for a new microservice. What factors should I consider?"
→ Model lists latency, tooling, team familiarity, ecosystem support

Q: "For a team of 5 junior developers, which would you recommend and why?"
→ Model factors in learning curve and debugging ease

Q: "What are the common mistakes teams make when adopting gRPC?"
→ Model surfaces proto versioning, error handling, and load balancing pitfalls
```

## Common Pitfalls
- **Over-interrogation**: Asking too many questions without acting — analysis paralysis
- **Assuming model authority**: LLMs can be confidently wrong — verify critical information
- **Context window waste**: Long interrogation chains consume tokens — summarize and consolidate
- **Missing the imperative phase**: Interrogation is for knowledge gathering, not implementation
